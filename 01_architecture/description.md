Название проекта: Интеллектуальная система диалогового оценивания знаний и управления пространствами кампуса.

Программный комплекс объединяет два направления: интеллектуальное оценивание знаний студентов и управление пространствами кампуса на базе компьютерного зрения. В состав решения входят Telegram-бот, WEB AI Tutor, админ-панель, ИИ-сервер, Java Core Backend и Python Web Backend, а в пользовательской модели выделены преподаватели и студенты.

Образовательный модуль заменяет классическое тестирование на диалоговое оценивание, где используются сократический диалог, дебаты и метод Фейнмана, а глубина понимания определяется по таксономии Блума. Преподаватель получает карту понимания студентов и может просматривать результаты по группам.

Модуль компьютерного зрения использует существующие камеры видеонаблюдения для определения занятости переговорных, рабочих мест, парковок и других зон, строит тепловые карты и прогнозирует загруженность. Решение ориентировано на внедрение в уже существующую инфраструктуру образовательного учреждения.

Telegram-бот предназначен для быстрого доступа к пространствам, проверяет роль пользователя, поддерживает бронирование и отмену броней, а также учитывает ограничения доступа и кулдауны. Админ-панель используется для настройки системы, управления лицензиями и загрузки расписаний.

## Подробное техописание

Система реализуется как локально разворачиваемый программный комплекс для одного образовательного учреждения. Архитектурно решение разделено на клиентские интерфейсы, Java Core Backend, Python Web Backend, ИИ-сервисы и подсистему хранения данных.

Java Backend является центральным слоем бизнес-логики и главным backend-ядром системы. Он отвечает за авторизацию, проверку ролей, хранение пользователей, групп, расписаний, пространств, броней, лицензий и правил доступа.

Python Backend не является core-слоем и используется только как специализированный web/AI сервис. Он обслуживает AI Tutor, Telegram webhook и интеграцию с отдельным ИИ-сервером.

ИИ-сервер выделяется как отдельный вычислительный контур, так как он отвечает за оценку ответов студентов и генерацию вопросов в сценарии диалогового оценивания. Модуль CV/Камеры отдельно подключает RTSP-потоки, запускает обработку через YOLO и обновляет данные о занятости пространств.

## Модули системы

- Telegram-бот. Обеспечивает сценарии первичной активации пользователя, проверки роли, отображения доступных пространств, создания брони и отмены брони.
- WEB AI Tutor. Реализует проведение диалогового оценивания и работу студента с AI Tutor через web-интерфейс.
- Java Core Backend. Реализует основную бизнес-логику, авторизацию, доступ к БД и API для frontend и admin-panel.
- Python Web Backend. Реализует AI Tutor API, webhook для Telegram и интеграцию с `ai-server`.
- Модуль CV/Камеры. Обрабатывает видеопотоки, определяет занятость зон и обновляет статусы пространств.
- Админ-панель. Используется для входа администраторов, управления лицензиями, whitelist и загрузки расписаний.

## Авторизация и доступ

Для админ-панели и пользовательских web-интерфейсов используется централизованная авторизация через Java Backend. Именно Java Core проверяет существование пользователя, его роль, пароль и выдает JWT-токены для доступа к защищенным API.

Python Backend не реализует отдельный независимый контур общей авторизации. Он использует данные и права доступа, полученные от Java Core.

## Лицензирование и активация

Система поддерживает раздельную активацию продуктов по ключам: отдельно для модуля AI Tutor и отдельно для модуля камер/CV. Управление лицензиями выполняется через административный контур, опирающийся на Java Backend как основной источник бизнес-данных.

## Загрузка расписания

В административном модуле реализован flow загрузки расписания, где frontend-часть отвечает за выбор и отправку Excel-файла, а Java Backend — за построчную обработку, проверку коллизий и транзакционное сохранение данных. При наличии ошибок сохранение отменяется, а администратору возвращается структурированный перечень проблемных строк.

## Диалоговое оценивание

В AI Tutor студент проходит не обычный тест, а диалоговую сессию, в которой ИИ задает вопросы, анализирует ответы и меняет стратегию: уточняет ответ, просит объяснить простыми словами или переходит в режим оппонента. Web-часть сценария обслуживается Python Backend, а основная академическая и пользовательская модель остается согласованной с Java Core.

## Компьютерное зрение

Модуль CV принимает видеопоток с камеры, выполняет детекцию объектов через YOLO, применяет правила зоны и обновляет статус пространства как занятого или свободного. На базе этих данных формируются статусы, аналитика и механизмы виртуального бронирования.
